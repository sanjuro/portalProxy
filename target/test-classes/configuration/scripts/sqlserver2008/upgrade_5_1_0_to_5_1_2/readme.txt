The database migration from 5.1.0 to 5.1.2 involves running the following steps in the order mentioned:

1. Create a backup of the database.

Run these steps before the new Portal Web Application has been deployed!
2. Run pre-dml-upgrade-5.1.0-to-5.1.2.ddl

If step 2 fail contact Backbase Support.
If you need to rollback the migration install the database backup.
